# Security Policy

## Supported versions

This project is maintained as a small operational repository. Security fixes are most likely to land on the current default branch.

## Reporting a vulnerability

Please **do not open a public issue with exploit details**.

If GitHub private vulnerability reporting is enabled for this repository, use that channel.

If private reporting is not enabled yet:

- open a minimal issue without sensitive details only if you have no private contact path available, or
- contact the maintainer through a private GitHub channel if one is available from the profile or repository settings.

When reporting, include:

- affected file or component
- impact summary
- reproduction steps or proof of concept
- any suggested mitigation

## Sensitive data reminder

Because this repository relates to DICOM/Horos workflows, never include:

- patient-identifying data
- real study metadata
- unredacted logs
- screenshots with names, dates, accession numbers, or UIDs

Please sanitize examples before sharing.
